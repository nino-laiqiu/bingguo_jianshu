2.where型：

把内层查询的结果作为外层查询的比较条件。比如查询最大商品 最贵商品

select goods,goodsname from tablename where goods ＝(select max (goods) from tablename)

如果where列＝SQL内嵌 这内嵌SQL返回必是单列 单个值

如果where列 in （内嵌SQL），则SQL只返回单列 可以多行


3.from查询


内层SQL的查询结果，当成一张临时表，供外层SQL，再次查询


4.exists 子查询

把外层的查询结果拿到内层，看内层的查询是否成立。 例如查询商品的栏目


5.为什么建标时，加not null default ''/default 0

不想让表中出现null null是一种类型 比较时只能用Is null  Is not null 碰到运算符一律返回null 效率不高
