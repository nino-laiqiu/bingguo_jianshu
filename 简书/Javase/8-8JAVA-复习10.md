#1.StringBuffer/StringBuilder构造方法 成员方法
 StringBuilder 线程不安全的(效率高)             StringBuffer: 线程安全的
 String: 每次更改,创建新的对象,性能低         StringBuilder: 不需要创建新的,效率高

###构造方法
public StringBuffer() 
public StringBuffer(int capacity)
public StringBuffer(String str)
```
public class Demo3 {
    public static void main(String[] args) {
        StringBuffer stringBuffer = new StringBuffer();
        //System.out.println(stringBuffer);//空
        //System.out.println(stringBuffer.capacity());//原来里面就有16个字符
        StringBuffer stringBuffer1 = new StringBuffer(10);
        System.out.println(stringBuffer1);//空
        System.out.println(stringBuffer1.length());//0
        System.out.println(stringBuffer1.capacity());//10  capacity是容量为10
        StringBuffer stringBuffer2 = new StringBuffer("12345");
        System.out.println(stringBuffer2.capacity());//21   原来是16 加 5
        System.out.println(stringBuffer2.length());//5 长度
        stringBuffer2.append("11111");
        System.out.println(stringBuffer2.capacity());//21 原来有16个空的字符 "11111"要填补16空值
    }
}
```

class Demo2 {
    public static String get(int[] array) {


    StringBuffer sb = new StringBuffer();//调用这个类
         sb.append("[");
        for (int i = 0; i < array.length; i++) {
            if (i == array.length -1){
                sb.append(i + "]");
            }
            else {
                sb.append(i + ",");
            }
        }
        return sb.toString();//调用这个方法把类型转为string类型
}
