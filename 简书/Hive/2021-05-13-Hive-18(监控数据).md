##1. 需求
一个需求:我需要知道SQL查询的结果是否有数据,并且要知道最近10天内是否有无数据
采取的方法是把所有的SQL union all 一起做一个job,当然这里需要考虑表(SQL)的复用性问题,我们需要的每天的情况,所以我们不需要按照原来的逻辑来写SQL,需要对SQL进行加工,来得到每天至少一条数据,如果连一条数据都没有那这一天肯定没有数据,这里涉及的优化有 表的复用性和union all 的优化事实上也没必要有什么优化,SQL的逻辑很简单只是count一下

如何判断是否有数据,如果能查询到就说明有数据,但是这么多的数据不容易看出来是否有数据,因为一般是有数据的,除非任务失败等非常规任务,所以我们取它们的反面,构建一张维度表与这些SQL left join

这里维度表的设计就是一个难点,怎么设计出只返回真正的无数据的天数,这样的化后台就不需要再次加工逻辑了

##2. 复用性问题
```
SELECT 
      ${group_col_1111},
      ' ..... ' as '接口名字',
      cnt
FROM
    (SELECT 
       ${group_col_1111},cnt,ROW_NUMBER() OVER (partition by d order by cnt) AS rn
    FROM 
        (SELECT 
            ${group_col_1111},count(1) as cnt
        FROM tabe1 
        WHERE d >= '${pre10_date}'
          AND d <= '${pre1_date}'
          AND .....
          AND city  in (${to_city_name})
          AND resident_prov_name IS NOT NULL
        GROUP BY ${group_col_1111}
        ) t1
    )t2
WHERE rn=1
```
上面的逻辑是取每天一条数据,但是与实际查的情况不一致,然后检查了好久,这里的参数是  city  in (${to_city_name}) 这里有就多个城市选取最大的那个,所以会返回数据...这是错误的写法,只能是一个city ,但是这样的SQL的复用性就降低了...有什么好的方法
