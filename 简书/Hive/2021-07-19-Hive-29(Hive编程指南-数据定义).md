##1. hive中的数据库
```sql
create database id not exists fina
```
如果数据库非常多,可以使用正则表达式来筛选需要的数据库名
```
hive> show databases like 'h.*'
```
load数据库的绝对路径和相对路径

默认情况下, Hive是不允许用户删除一个包含有表的数据库的。用户要么先删除数据库中的表,然后再删除数据库;要么在删除命令的最后面加上关键字CASCADE

##2. 修改数据库
没有办法可以删除或者“重置”数据库属性
##3. 创建数据库
默认情况下, Hive总是将创建的表的目录放置在这个表所属的数据库目录之后。不过,default数据库是个例外,其在/user/hive/warehouse下并没有对应一个数据库目录。因此default数据库中的表目录会直接位于/user/hive/warehouse目录之后(除了用户明确指定为其他路径）
为了避免潜在产生混淆的可能性,如果用户不想使用默认的表路径,那么最好是使用外部表
