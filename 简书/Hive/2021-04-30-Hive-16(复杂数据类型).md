##1.复杂数据类型
#####array map struct
```
注意array 和类型struct的区别array中只能存放同一种数据类型,而struct能存放不同的数据类型

subordinates	array<string>, 
deductions	    map<string,float>, 
address	        struct<street:string, city:string, state:string, zip:int>

三者的区别如下
array中只能装同一种数据类型,如下访问

with x as (
 select array('1','2','3') as arr 
)
select arr[0] from x --> 1 

对于map类型中,存放固定的两种类型,与struct的区别在于struct中的key键是固定的

SELECT t['A']
FROM (
    SELECT map('A', 10, 'B', 20, 'C', 30) AS t
) t1
struct类型的使用,这里好像不能使用key值,这么是默认的key值
select *  from ( select struct('B',20,date '2016-02-02') as t ) t1
```
注意一下,如何访问,有什么区别即可,注重实际的解析功能

#####inline函数
```
select col1,col2,col3  from (select 0) t lateral view inline(array(struct('A',10,date '2015-01-01'),struct('B',20,date '2016-02-02'))) tf as col1,col2,col3;
```
#####json_tuple()
解析json,比get_json_object()高效一点,对比
```
select get_json_object(line,'$.movie') as movie,
	get_json_object(line,'$.rate') as rate,
	get_json_object(line,'$.timeStamp') as time,  
	get_json_object(line,'$.uid') as uid
from rate_json;


select b.movie,b.rate,b.time,b.uid
from rate_json
lateral view json_tuple(line,'movie','rate','timeStamp','uid') b
as movie,rate,time,uid;
```

##2.自定义UDF
>https://cwiki.apache.org/confluence/display/Hive/LanguageManual+UDF
#####map_to_json
对map中的key排序,生成json格式,调用ap即可
#####list_merge
合并两个list,去除重复的元素,开发很简单就是两个list装入一个hashset中
#####list_filter
去重某个字符
#####list_agge
合并多个list
#####map_filter
传入一个array和一个map,选出map中的key在这个array中的值,返这个包含的map
等等这些自定义的UDF数仓中没啥用,大多用在ABT/数据治理UDT中
#####拼接json
就是拼接json的key和values,注意传入的参数必须是双数,判断一下
