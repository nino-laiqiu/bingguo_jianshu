##1.去重
传统的去重做法,虽然底层会进行谓词下推的优化,但是效率还是不高
```
select count(distinct)
```
去重的优化,对维度分步骤group by 
```
select a,b,count(c)
from table1
(
select a,b,c
from table1
group by a,b,c
)
group by a,b
```
去重是大数据技术中的一个难点,最近我在学习的kylin中就有应用(计数器),具体的去重方法还是要结合具体的业务,总之是一个难点
##2.join
1. 不要在on条件使用函数

错误的写法
```
join .... on substr(trim(t1.a),1,2) = substr(trim(t2.a),1,2) 
```
正确的写法,在select中进行转换,之后再join
```
select substr(trim(t1.a),1,2) 
```
2. 过滤不需要的条件

尽量select中过滤所需要的的分区和字段
```
(select a,b where t = '2021-01-01) join (select a,b where t = '2021-01-02)
```

由于某些字段可能非常大,导致某个reduce数据过多,内存问题影响速度等,优化方法有:
1. 大key和小key union all 
2. 负载均衡
##6.map端join
map端join类似于spark广播变量的形式,把小表加载到内存中,这样就避免了对reduce的压力以及解决数据倾斜,我们需要查找出那些top_key以及测试top_key join的性能,一般10M的数据底层就自动为mapjoin,因为mapjoin性能较好,所以可以设置参数 spark.sql.autoBroadcastJoinThreshold = ? 来提高mapjoin的数据量级
