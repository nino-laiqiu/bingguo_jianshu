##1.Project 和 Module 介绍
>一个 Project 是由一个或多个 Module 组成，模块之间尽量是处在同一个项目业务的的情况下，彼此之间互相依赖关联。这里用的是 尽量，因为 IntelliJ IDEA 的 Project 是一个没有具备任何编码设置、构建等开发功能的，主要起到一个项目定义、范围约束、规范等类型的效果，也许我们可以简单地理解为就是一个单纯的目录，只是这个目录命名上必须有其代表性的意义。


##2.快捷键
1.Ctrl + h  查看子类
2.alt + 7  参看类中的方法
3.shift + enter 切换到下一行
4.home 到行的头 end 到行的尾
5.Ctrl + F12 查找类中的方法
6.Ctrl + 7 查看类中的方法
7.Ctrl + p  查看要传入的参数


>Ctrl    +  F 在当前文件进行文本查找
>Ctrl   +  R 在当前文件进行文件替换

![image.png](https://upload-images.jianshu.io/upload_images/9049859-0fcdd65f399b5a94.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)


>Ctrl + Y    删除光标所在行 或 删除选中的行 （必备）
  Ctrl + X	剪切光标所在行 或 剪切选择内容
  Ctrl + C	复制光标所在行 或 复制选择内容


>Ctrl + N	根据输入的 类名 查找类文件 （必备）
  Ctrl + E	显示最近打开的文件记录列表 （必备）


>Ctrl + J	插入自定义动态代码模板 （必备）
>Ctrl + P	方法参数提示显示 （必备）


>Ctrl + F1	在光标所在的错误代码处显示错误信息 （必备）
  Ctrl + F3	调转到所选中的词的下一个引用位置 （必备）


>Alt + Enter	IntelliJ IDEA 根据光标所在问题，提供快速修复选择，光标放在的位置不同提示的结果也不同 （必备）

>Ctrl + Alt + S	打开 IntelliJ IDEA 系统设置 （必备）

>Ctrl + Shift + T	对当前类生成单元测试类，如果已经存在的单元测试类则可以进行选择 （必备）


##2.debug的技巧与快捷键

>F7:进入下一步，如果当前行断点是一个方法，则进入当前方法体内，如果该方法体还有方法，则不会进入该内嵌的方法中 必备

>F8:进入下一步，如果当前行断点是一个方法，则不进入当前方法体内 必备

>F9:恢复程序运行，但是如果该断点下面代码还有断点则停在下一个断点上 必备

>ALT + F8  选中对象，弹出可输入计算表达式调试框，查看该输入内容的调试结果 必备
