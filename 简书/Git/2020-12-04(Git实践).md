##1.
> git commit  提交新的记录

##2.
> git branch newImage 创建新分支
git commit 提交当前分支

> git branch newImage ; git commit 切换到新的分支,并且把新的记录提交到新分支


##3.
分支的合并
例如我们把bugFix 合并到master
> git merge bugFix

如下:把master合并到bugFix
> git  checkout bugFix ; git merge  master 切换到 bugFix 合并master到bugFix

> git rebase master 
把bugFix 分支里的工作直接移动到master分支上,但是master还没有更新

> git rebase bugFix 
由于bugFix继承自master,所以git只是简单的把master分支的引用向前移动了一下而已

> git revert HEAD
在我们要撤销的提交记录后面多了一个新提交！这是因为新提交记录 C2' 引入了更改 这些更改刚好是用来撤销 C2 这个提交的 也就是说 C2' 的状态与 C1 是相同的
